Wi-Fi

Отправляете другу «Привет» — на экране 6 букв. Но внутри телефона букв нет: только электрические сигналы в двух состояниях — «вкл» и «выкл».

Бит (bit, binary digit) — один сигнал: 0 или 1. Один ответ «да/нет», один переключатель.

Кто решил, что «П» = такой набор бит? Люди договорились. Как ценник «20 руб.» понятен всем — есть таблицы кодирования, связывающие символы с числами, а числа — с битами.

0.2a · Бит и байт

8 переключателей = 1 байт

Каждый бит — переключатель с весом. Включённые складываем:

1 бит → 2 значения

да / нет, вкл / выкл

1 байт = 8 бит

0–255: символ, RGB-компонента, отсчёт

2⁸ = 256

комбинаций. 255 = 2⁸−1 — максимум байта

Кракозябры → UTF-8

Проблема 90-х

Код 192 в КОИ-8 — одна буква, в Windows-1251 — другая. Письмо в чужой кодировке → каша.

«Привет» → «╧ЁштхЄ»

Решение: UTF-8

Один стандарт для всех языков. ASCII (0–127) = 1 байт, кириллица = 2, иероглифы = 3, эмодзи = 4.

● ● ●«Привет» в UTF-8□ _ ✕

Unicode: 154 000+ символов. 98 % веба — UTF-8. Цена универсальности.

0.4 · Фото

Фото = сетка пикселей

Фотография разбивается на пиксели. Каждый пиксель — три числа R, G, B (0–255 = 1 байт). Кликни по фото или сетке — выбери пиксель; кнопки ниже запускают авто-обход: фото → сетка → пиксель → код.

клик по ① фото или ② сетке — выбрать пиксель

FFFFFF = белый, 000000 = чёрный, FFFF00 = жёлтый, FF0000 = красный. Так задают цвет в HTML/CSS: #RRGGBB.

12 Мп = 4000×3000 = 12 млн пикс
× 3 байта = 36 МБ один снимок
JPEG сжимает ≈ ×10 → 3,4 МБ
Как это работает Матрица камеры меряет яркость света → АЦП превращает в число. Фото — это прямоугольная таблица чисел, и по сети идёт как последовательность нулей и единиц.
0.5 · Звук

Звук = отсчёты во времени

Микрофон превращает колебания воздуха в напряжение. АЦП измеряет его N раз в секунду и округляет до ближайшего уровня → поток чисел.

fд × разрядность × каналы = поток
Телефон: 8000 × 8 × 1 = 64 кбит/с
CD: 44100 × 16 × 2 = 1411 кбит/с
Теорема Котельникова Чтобы восстановить сигнал, fд ≥ 2·fmax. Ухо слышит до 20 кГц → CD берёт 44,1 кГц. MP3/Opus сжимают в 10 раз.
0.6 · Видео

Видео = 25 картинок/сек

Видео — это последовательность кадров, каждый кадр = фотография. Меняются так быстро, что глаз видит движение.

1 кадр Full HD = 1920×1080×3 = 5,93 МБ
× 30 fps = 178 МБ/с · 1 мин = 10,7 ГБ
без сжатия передать невозможно

Секрет сжатия: соседние кадры почти одинаковы. Кодируем не каждый кадр, а разницу.

I-frame — опорный (полный JPEG). P/B-frame — только изменения. H.264 → H.265 → AV1, каждый на 30–50 % лучше.

Итог: фильм 1,5 ч сжимается с ~950 ГБ до 4–8 ГБ. Поток 1080p ≈ 5 Мбит/с.

0.7 · Counter-Strike

Игра = быстрая передача чисел

Онлайн-игра — это быстрая передача огромного количества данных. Сервер хранит «истинный» мир, клиент шлёт ваши действия и рисует картинку.

● ● ●пакет клиента → сервер□ _ ✕
Номер тика 4 байта (uint32) Координаты прицела 8 байт (2×float32) Нажатые клавиши 2 байта (битовая маска) Активное оружие 1 байт ──────────────────────── Итого: ~20 байт + заголовки

Каждый кусок кодируется своим числом и шлётся 64–128 раз в секунду.

~80 байт × 128 тик ≈ 82 кбит/с
меньше мегабита — в 10–60 раз меньше видео

Картинку рисует ваша видеокарта; по сети идут координаты и события. Клавиши = битовая маска: бит 0 — вперёд, бит 7 — стрельба.

0.8 · Системы счисления

Цифра ≠ число

Цифра

Символ для записи. В DEC: 0–9 (их 10). Набор конечен.

Число

Количество чего-либо. Чисел бесконечно.
Двадцать = 20₁₀ = 10100₂ = 14₁₆ = XX

Позиционная запись: позиция цифры задаёт её вес.

356 = 3×10² + 5×10¹ + 6×10⁰ = 300+50+6
1101₂ = 1×2³+1×2²+0+1×2⁰ = 8+4+1 = 13
F4₁₆ = 15×16¹+4×16⁰ = 240+4 = 244
356₈ = 3×64+5×8+6 = 238

0.8 · Три системы счисления

DEC · BIN · HEX

Одно число — разные записи. Основание = сколько цифр в системе и во сколько раз тяжелеет разряд влево.

DEC · основание 10

Цифры 0–9. Разряд левее — ×10. Ценники, счёт.

365 = 3·10² + 6·10¹ + 5·10⁰ = 365

BIN · основание 2

Цифры 0 и 1 (выкл/вкл). Разряд — ×2. Биты, маски.

1101₂ = 8 + 4 + 0 + 1 = 13₁₀

HEX · основание 16

Цифры 0–9, A–F (A=10…F=15). Разряд — ×16. Цвета, дампы.

D2₁₆ = 13·16 + 2 = 210₁₀
Одно число — три записи 210₁₀ = 1101 0010₂ = D2₁₆ — один и тот же байт. 2 hex-цифры = 1 байт (4 бита на цифру). Дальше счётчик +1 покажет перенос в каждой системе.
0.8 · Переполнение

Барабан крутится

Последняя цифра → перенос в старший разряд. DEC: 9→10. BIN: 1→10. HEX: F→10. Смотрите, как счётчик идёт +1 → жёлтым подсвечены биты, которые переключились (перенос).

0.9 · Перевод

DEC → BIN: деление на 2

Делим на 2, записываем остаток, читаем снизу вверх.

● ● ●73₁₀ → ?₂□ _ ✕
73 ÷ 2 = 36 ост. 1 36 ÷ 2 = 18 ост. 0 18 ÷ 2 = 9 ост. 0 9 ÷ 2 = 4 ост. 1 4 ÷ 2 = 2 ост. 0 2 ÷ 2 = 1 ост. 0 1 ÷ 2 = 0 ост. 1 ↑ Читаем: 1001001₂

Зачем связисту:

BIN — язык аппаратуры

HEX — компактная запись

DEC — повседневность

Быстрый способ 150 = 128+16+4+2 = 2⁷+2⁴+2²+2¹ → 10010110₂. Проверка: 64+8+1 = 73 ✓

Тот же алгоритм с делением на 16 → перевод в HEX. Скоро запрограммируем его.

0.9 · Тетрады

BIN ↔ HEX: 4 бита = 1 hex-цифра

16 = 2⁴ → каждая hex-цифра = ровно 4 бита. Как тетрис: кубики «щёлкают» по четыре.

F₁₆ = 1111₂    4₁₆ = 0100₂
F4₁₆ = 1111 0100₂

Обратно: разбиваем справа по 4:

0100 1001 → 4 9 → 49₁₆
4×16+9 = 73 ✓

Аналогично 8 = 2³ → триады (BIN ↔ OCT):

3₈=011 5₈=101 7₈=111
357₈ = 011101111₂

0.1 · Бутерброд с вареньем

Декомпозиция: разбей на шаги

СЦЕНКА

Банка варенья, хлеб, ложка. Опиши рецепт для робота-повара — он понимает только точные команды.

● ● ●recipe.sh — идеальный алгоритм□ _ ✕
1. Открыть хлебницу 2. Достать один ломтик хлеба 3. Положить на разделочную доску 4. Открыть банку с вареньем 5. Взять чайную ложку 6. Набрать полную ложку варенья 7. Намазать на верхнюю сторону 8. Положить бутерброд на тарелку 9. Закрыть банку

Большая задача → маленькие шаги, каждый = одна команда. Компьютер педантичен: не додумает, не переспросит.

Пропустили шаг 4? Робот набирает варенье из закрытой банки → ошибка.

Перепутали 7 и 8? Варенье на тарелке, а не на хлебе.

Зачем связисту Настройка роутера — тот же рецепт: создать интерфейс → назначить IP → поднять протокол → разрешить трафик. Пропустил шаг — связь не встанет.
0.1 · Определения

Алгоритм ≠ программа

● ● ●алгоритм□ _ ✕

Последовательность шагов для решения задачи, записанная на человеческом языке, блок-схеме или псевдокоде.

Не зависит от языка. Один алгоритм → Python, Си, Java.

● ● ●программа□ _ ✕

Алгоритм, записанный на конкретном языке программирования и готовый к выполнению.

Зависит от языка: одна задача на Python и Си выглядит по-разному.

Аналогия Рецепт блюда — это алгоритм. Тот же рецепт на конкретном языке для конкретного повара — программа. Блюдо одно.

0.1б · Линейный алгоритм

Бутерброд = конвейер

Все шаги идут последовательно, без ветвлений и повторов. Стрелка — только вниз.

На практике Чистая линейность редка — обычно есть хотя бы одно условие. Линейные участки — «прямые» между ветвлениями.

Нужен второй ломтик? Добавь ещё один прямоугольник перед «Конец». Нужна проверка «есть ли варенье?» — нужен ромб (следующий слайд).

0.1б · Ветвление

Ромб: чётное или нечётное?

Из ромба выходят два пути: «Да» и «Нет». Каждый ведёт к своему действию, потом они сходятся.

● ● ●python□ _ ✕

Теорема Бёма–Якопини (1966) Любой алгоритм = последовательность + ветвление + цикл. Других конструкций не нужно.
0.1б · Цикл

Стрелка назад = цикл

Возвратная стрелка возвращает поток к проверке. Тело повторяется, пока условие истинно.

Когда N станет нулём — условие ложно, цикл завершается. Это и есть перевод в двоичную делением на 2.

● ● ●python□ _ ✕
while n > 0: бит = n % 2 n = n // 2
Соответствие ромб → while, прямоугольник → присваивание, стрелка назад → закрывающая } цикла.

0.10 · Mathcad — операторы

Два знака равенства: := и =

:=  «задать значение»

Набираешь : (двоеточие) — Mathcad сам превращает в :=. «Запомни, что a равно пяти».

=  «покажи результат»

Mathcad подставляет и считает. «Чему сейчас равно?»

● ● ●лист Mathcad□ _ ✕
a := 5
b := a² + 3
b = 28
Магия пересчёта Меняешь a := 7 — и весь лист пересчитывается: b автоматически станет 52. В математике := и = — одно, в Mathcad их различают.

0.10 · Python — цикл while

while: пока условие истинно

for — когда знаем, сколько раз. while — когда повторяем «пока выполняется условие».

● ● ●countdown.py□ _ ✕
n = 5 while n > 0: print(n) n = n - 1 # 5, 4, 3, 2, 1
● ● ●та же сумма через while□ _ ✕

Осторожно Если условие никогда не станет ложным — бесконечный цикл. Внутри тела должно меняться то, что в условии. while — основа нашей программы перевода в двоичную.

0.10 · Python — первая программа

to_binary(): DEC → BIN

ЗАДАЧА

Перевести десятичное число в двоичную запись делением на 2 с остатком (как в §0.9 на бумаге).

① Блок-схема

② Программа

● ● ●to_binary.py□ _ ✕
def to_binary(n): if n == 0: return '0' bits = [] while n > 0: bits.append(n % 2) # остаток n = n // 2 # сдвиг → bits.reverse() return ''.join(str(b) for b in bits) print(to_binary(73)) # 1001001 print(to_binary(255)) # 11111111
Встроенная bin(73) → '0b1001001'. Написав свою — понимаешь, как она работает.
0.10 · Си

Hello World + компиляция

● ● ●hello.c□ _ ✕
#include <stdio.h> int main(void) { printf("Привет, связь!\n"); return 0; }
gcc hello.c -o hello → ./hello
  • #include — подключить библиотеку
  • int main — точка входа
  • printf — напечатать. \n = LF (код 10)
  • return 0 — «всё ОК»
  • { } — границы блока (вместо отступов)
  • Каждая команда — ;

Python = устный переводчик (выполняет на лету). Си = письменный: сначала весь текст переводит компилятор (gcc), потом читаете готовое.

0.10 · Си vs Python

DEC → BIN: бок о бок

ЗАДАЧА

Тот же перевод в двоичную — блок-схема та же (деление на 2 с остатком), меняется только язык программы.

● ● ●python · 8 строк□ _ ✕
def to_binary(n): if n == 0: return '0' bits = [] while n > 0: bits.append(n % 2) n = n // 2 bits.reverse() return ''.join(str(b) for b in bits)
● ● ●c · 15 строк□ _ ✕
#include <stdio.h> int main() { int n; char buf[33]; scanf("%d", &n); int i = 0; while (n > 0) { buf[i++] = n%2 + '0'; n /= 2; } for (int j=i-1; j>=0; j--) putchar(buf[j]); putchar('\n'); }
Итог Код длиннее, но быстрее и ближе к железу. Тот же алгоритм — другой уровень контроля.
0.10 · Шеннон–Хартли

Одна задача — три решения

ЗАДАЧА

C = B · log₂(1 + SNR). Канал B = 3100 Гц, SNR = 30 дБ. Найти предельную пропускную способность.

● ● ●Mathcad□ _ ✕
B := 3100
SNRdB := 30
SNR := 10^(SNRdB/10)
C := B·log(1+SNR, 2)
C = 30 899
● ● ●Python□ _ ✕
import math B = 3100 SNR = 10 ** (30/10) C = B * math.log2(1+SNR) print(f"C={C:.0f}") # C=30899
● ● ●Си□ _ ✕
#include <math.h> double B=3100; double S=pow(10,30/10.0); double C=B*log2(1+S); printf("%.0f",C); // 30899 (gcc -lm)
Три программы — одно число C ≈ 30 899 бит/с ≈ 31 кбит/с — предел телефонного канала. Модем V.34 достигал 33,6 кбит/с — почти потолок Шеннона. Совпали — расчёт верен.
